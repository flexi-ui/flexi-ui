# Next.js

FlexiUI works with Next.js 14+ (App Router or Pages Router). The setup is a single CSS import — no plugins, no `tailwind.config.ts` globs into `node_modules`, no provider.

## 1. Create or open a Next.js project

```bash
pnpm create next-app@latest my-app
cd my-app
```

Pick "Yes" for TypeScript and Tailwind CSS when the CLI prompts. FlexiUI requires Tailwind v4, which is the default in current versions of `create-next-app`.

## 2. Install FlexiUI

```bash
pnpm add @flexi-ui/styles @flexi-ui/button
```

Add more component packages as you need them (`@flexi-ui/input`, `@flexi-ui/card`, etc.).

## 3. Import the stylesheet

Replace the contents of `app/globals.css`:

```css
/* app/globals.css */
@import '@flexi-ui/styles/css';

@custom-variant dark (&:where(.dark, .dark *));

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
}
```

`@flexi-ui/styles/css` already imports Tailwind internally — don't add another `@import 'tailwindcss'`.

Make sure `app/layout.tsx` imports this file:

```tsx
import './globals.css'
```

## 4. Use a component

```tsx
// app/page.tsx
import { Button } from '@flexi-ui/button'

export default function Home() {
  return (
    <main className="flex min-h-screen items-center justify-center">
      <Button>Hello FlexiUI</Button>
    </main>
  )
}
```

Run `pnpm dev` — that's the full setup.

## Client-side routing

To make `<Link>` (and any other `href`-aware component) navigate with `next/navigation` instead of reloading the page, wrap your app in React Aria's `RouterProvider`. See the [Routing guide](/docs/guide/routing) for the exact pattern.

## Dark mode

FlexiUI scopes its dark-mode rules to `.dark` on any ancestor. Pair it with a library like [`next-themes`](https://github.com/pacocoursey/next-themes) to toggle the class:

```tsx
// app/providers.tsx
'use client'

import { ThemeProvider } from 'next-themes'

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider attribute="class" defaultTheme="system">
      {children}
    </ThemeProvider>
  )
}
```

```tsx
// app/layout.tsx
import { Providers } from './providers'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <Providers>{children}</Providers>
      </body>
    </html>
  )
}
```

## Server vs client components

FlexiUI components use React state and event handlers, so they're client components. Import them inside files with `'use client'` at the top, or drop them into any existing client subtree — you don't need to mark every page as client, just the parts that actually render interactive components.
