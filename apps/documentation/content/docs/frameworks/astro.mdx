# Astro

Astro renders static HTML by default, so FlexiUI components (which are React client components) need Astro's React integration and a `client:*` directive on each island.

## 1. Create or open an Astro project

```bash
pnpm create astro@latest my-site
cd my-site
```

## 2. Add React and Tailwind v4

```bash
pnpm astro add react
pnpm add -D tailwindcss @tailwindcss/vite
pnpm add @flexi-ui/styles @flexi-ui/button
```

## 3. Register the Tailwind Vite plugin

```ts
// astro.config.mjs
import { defineConfig } from 'astro/config'
import react from '@astrojs/react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  integrations: [react()],
  vite: {
    plugins: [tailwindcss()],
  },
})
```

## 4. Create the stylesheet

```css
/* src/styles/global.css */
@import '@flexi-ui/styles/css';

@custom-variant dark (&:where(.dark, .dark *));

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
}
```

Import it from your layout or root page:

```astro
---
// src/layouts/Layout.astro
import '../styles/global.css'
---
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My site</title>
  </head>
  <body>
    <slot />
  </body>
</html>
```

## 5. Use a FlexiUI component as an island

FlexiUI components are interactive, so Astro needs a hydration directive:

```astro
---
// src/pages/index.astro
import Layout from '../layouts/Layout.astro'
import { Button } from '@flexi-ui/button'
---
<Layout>
  <main class="flex min-h-screen items-center justify-center">
    <Button client:load>Hello FlexiUI</Button>
  </main>
</Layout>
```

Pick the directive that fits the component's role:

- `client:load` — hydrate immediately (good for above-the-fold interactive UI).
- `client:idle` — hydrate when the browser is idle.
- `client:visible` — hydrate when the island scrolls into view (ideal for footers, modals).
- `client:only="react"` — skip SSR entirely; only render on the client. Use this if a component depends on APIs that don't exist on the server.

## Dark mode

Toggle a `dark` class on `<html>` — FlexiUI's dark variant activates automatically. Since Astro doesn't ship with a theme library, use a small inline script in the `<head>` to apply the user's preference before paint, preventing a flash of unstyled content.
