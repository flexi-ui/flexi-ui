# Introduction

**FlexiUI** is a React component library you install one component at a time. Every component is built on [React Aria Components](https://react-spectrum.adobe.com/react-aria/) for accessibility, styled with [Tailwind CSS v4](https://tailwindcss.com/), and ships from its own package under `@flexi-ui/*`.

## Design goals

- **No provider required.** Drop a `<Button>` into any React tree — no `Provider` wrapper, no context shim.
- **Ship only what you use.** Each component is its own npm package, so your bundle carries `@flexi-ui/button` without pulling the rest of the library.
- **Accessibility by default.** React Aria handles keyboard, focus, screen reader semantics, and pointer interactions. FlexiUI composes these into a Radix-style compound API.
- **Themeable with CSS variables.** Colors, radii, and motion live in `@flexi-ui/styles` as CSS variables — override them in your own globals.

## Quick start

```bash
pnpm add @flexi-ui/styles @flexi-ui/button
```

```tsx
import { Button } from '@flexi-ui/button'

export default function App() {
  return <Button>Get started</Button>
}
```

Then import the stylesheet once, at the top of your app's root CSS:

```css
@import '@flexi-ui/styles/css';
```

That's the whole setup. See [Installation](/docs/guide/installation) for framework-specific wiring.

## What's in the box

- **~60 components** covering buttons, forms, navigation, data display, overlays, and more — browse the [Components](/docs/components/button) section for the full list.
- **A styles package** (`@flexi-ui/styles`) that ships the design tokens and base CSS. You import this once; the components read from it.
- **Shared hooks** under `@flexi-ui/use-*` for things like media queries and overlay state, if you want to build custom components on the same primitives.
