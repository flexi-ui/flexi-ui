# Tailwind v4

FlexiUI targets Tailwind CSS v4 exclusively. The styles package (`@flexi-ui/styles`) bundles its own `@import 'tailwindcss'` call, so installing FlexiUI is what bootstraps Tailwind in your app.

## What the styles package ships

`@flexi-ui/styles/css` is a single CSS entrypoint that loads, in order:

1. Tailwind v4 (`@import 'tailwindcss'`)
2. Base resets (`@layer base`)
3. BEM component styles for every FlexiUI component (`@layer components`)
4. Theme tokens — CSS variables for colors, radii, spacing (`@layer theme`)
5. Utilities and variants

You don't import these pieces individually — importing `@flexi-ui/styles/css` wires the whole stack.

## Project setup

### 1. Install the PostCSS plugin

```bash
pnpm add -D tailwindcss @tailwindcss/postcss
```

### 2. Enable the plugin

```js
// postcss.config.mjs
export default {
  plugins: ['@tailwindcss/postcss'],
}
```

### 3. Import the stylesheet

```css
/* styles/globals.css */
@import '@flexi-ui/styles/css';
```

Do **not** add a second `@import 'tailwindcss'` line — the styles package already loads it.

## Customizing tokens

FlexiUI's theme is exposed as CSS variables. Override them after the import to change colors, radii, or motion without touching Tailwind config:

```css
@import '@flexi-ui/styles/css';

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --accent: #4f46e5;
  --accent-foreground: oklch(1 0 0);
  --accent-muted: color-mix(in oklch, #4f46e5 14%, transparent);
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --accent: #818cf8;
}
```

The variables are consumed directly by the component BEM classes, so an override propagates to every `Button`, `Link`, `Input`, etc. without rebuilding.

## Dark mode

FlexiUI ships dark-mode rules scoped with:

```css
@custom-variant dark (&:where(.dark, .dark *));
```

Toggle the `dark` class on `<html>` (manually or via a library like `next-themes`) and the dark variant activates automatically.

## Mapping variables to Tailwind utilities

If you want `bg-background`, `text-foreground`, etc. as utility classes in your own code, map the CSS variables in a `@theme inline` block:

```css
@import '@flexi-ui/styles/css';

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-accent: var(--accent);
  --color-muted: var(--muted);
  --color-muted-foreground: var(--muted-foreground);
}
```

Tailwind v4 reads `@theme inline` to generate corresponding utilities. This is the same pattern the docs site itself uses — see the app's own `styles/globals.css`.

## Common pitfalls

- **Don't add a Tailwind plugin.** FlexiUI does not ship a Tailwind config plugin; everything it needs is in the CSS bundle.
- **Don't include FlexiUI source in your `content` globs.** v4's content detection is automatic; there's nothing to configure.
- **Order matters.** Put `@import '@flexi-ui/styles/css'` first in your globals file. Your overrides come after.
