# Installation

FlexiUI ships each component as its own package. You install `@flexi-ui/styles` once for the design tokens and base CSS, then add any components you actually use.

## Requirements

| Dependency   | Minimum version |
| ------------ | --------------- |
| Node.js      | 20              |
| React        | 19              |
| Tailwind CSS | 4               |

## 1. Add the styles package

```bash
pnpm add @flexi-ui/styles
```

```bash
npm install @flexi-ui/styles
```

```bash
yarn add @flexi-ui/styles
```

`@flexi-ui/styles` ships the CSS reset, component class rules, and the CSS variables the components read from at runtime. Every app needs it exactly once.

## 2. Import the stylesheet

Import the stylesheet from your app's root CSS entry point. In a Tailwind v4 project this file also contains your `@theme` block:

```css
/* styles/globals.css */
@import '@flexi-ui/styles/css';

@custom-variant dark (&:where(.dark, .dark *));

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  /* override FlexiUI tokens here */
}
```

`@flexi-ui/styles/css` resolves to the precompiled `index.css`. It already imports `tailwindcss` inside, so you do **not** need a separate `@import 'tailwindcss';` line.

## 3. Add components as you need them

Each component is a separate package. Install what you use:

```bash
pnpm add @flexi-ui/button @flexi-ui/input @flexi-ui/form
```

```tsx
import { Button } from '@flexi-ui/button'
import { Form } from '@flexi-ui/form'
import { Input } from '@flexi-ui/input'

export function SignIn() {
  return (
    <Form onSubmit={(e) => e.preventDefault()}>
      <Input label="Email" type="email" isRequired />
      <Input label="Password" type="password" isRequired />
      <Button type="submit">Sign in</Button>
    </Form>
  )
}
```

That's it — no `FlexiUIProvider`, no Tailwind plugin, no `content` globs pointing into `node_modules`.

## Why no provider?

Components style themselves from CSS variables defined in `@flexi-ui/styles`. There's no theme context to thread through the tree, so no provider is required. If you want dark mode, toggle a `dark` class on `<html>` — the stylesheet scopes dark overrides with `@custom-variant dark (&:where(.dark, .dark *))`.

## Troubleshooting

**Components render unstyled.** You forgot step 2 — the stylesheet import. Every BEM class (`.button--primary`, `.card__header`, etc.) lives in `@flexi-ui/styles/css`. Without that import, the JSX renders with empty class names.

**CSS variable values look wrong after a theme override.** Check that your `:root` block comes *after* the `@import '@flexi-ui/styles/css'` line. CSS cascades by document order; later rules win.

**`Cannot find module '@flexi-ui/button'`.** Install the specific component package (`pnpm add @flexi-ui/button`). There is no umbrella `@flexi-ui/react` package — every component is installed individually.
