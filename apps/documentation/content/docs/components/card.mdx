# Card

`Card` is a surface container used to group related content into a bounded region. It exposes a compound API — `Card.Header`, `Card.Body`, `Card.Footer` — so you can compose media, metadata, actions, and supporting copy with full control.

## Installation

```bash
pnpm add @flexi-ui/card @flexi-ui/styles
```

## Import

```tsx
import { Card } from '@flexi-ui/card'
```

## Anatomy

```tsx
<Card>
  <Card.Header />
  <Card.Body />
  <Card.Footer />
</Card>
```

| Part          | Element | Purpose                                    |
| ------------- | ------- | ------------------------------------------ |
| `Card`        | `div`   | Root surface. Applies size + variant.      |
| `Card.Header` | `div`   | Top slot for titles, media, or actions.    |
| `Card.Body`   | `div`   | Main content region.                       |
| `Card.Footer` | `div`   | Bottom slot for actions or meta. Supports `isBlurred`. |

## API Reference

### Card (Root)

| Prop          | Type                                              | Default     | Description                        |
| ------------- | ------------------------------------------------- | ----------- | ---------------------------------- |
| `variant`     | `'default' \| 'bordered' \| 'flat' \| 'shadow'`   | `'default'` | Surface style.                     |
| `size`        | `'sm' \| 'md' \| 'lg'`                            | `'md'`      | Padding + radius scale.            |
| `fullWidth`   | `boolean`                                         | `false`     | Stretch to container width.        |
| `isPressable` | `boolean`                                         | `false`     | Apply interactive affordances.     |

Extends native `div` props (omits `size`). The `size` prop is shared with descendants via context.

### Card.Header / Card.Body

Each accepts native `div` props — `className`, `children`, event handlers, etc.

### Card.Footer

Accepts native `div` props plus `isBlurred` (`boolean`, default `false`) to apply the frosted background treatment.

## Accessibility

- Cards are generic containers by default. If the card represents an interactive target (e.g. opens a detail view), wrap its content in a `Link` or `button` and include readable text.
- When using `isPressable`, make sure keyboard users can activate the card — attach `onClick` to a native interactive element or supply `role="button"`, `tabIndex={0}`, and an `onKeyDown` handler.
- Keep `Card.Header` content semantically accurate — if the title is a heading, render a proper `h2`/`h3` inside it rather than styling a `div`.
