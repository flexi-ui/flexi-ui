# Button

`Button` is the primary action trigger. It's built on top of React Aria Components' `Button` primitive, so it ships with keyboard handling, focus-visible rings, and press feedback for free.

FlexiUI folds tone and style into a single `variant` prop — there's no separate `color` prop. Ten variants cover the common combinations.

## Installation

```bash
pnpm add @flexi-ui/button @flexi-ui/styles
```

## Import

```tsx
import { Button, ButtonGroup } from '@flexi-ui/button'
```

## Usage

<Sandbox code={`<Button>Continue</Button>`}>

## Disabled

Use `isDisabled` (React Aria convention) rather than the native `disabled`. React Aria wires up the correct ARIA state and keyboard behavior.

<Sandbox code={`<div className="flex gap-3">
  <Button isDisabled>Disabled</Button>
  <Button isDisabled variant="bordered">Disabled</Button>
  <Button isDisabled variant="flat">Disabled</Button>
</div>`}>
  <div className="flex gap-3">
    <Button isDisabled>Disabled</Button>
    <Button isDisabled variant="bordered">Disabled</Button>
    <Button isDisabled variant="flat">Disabled</Button>
  </div>
</Sandbox>

## ButtonGroup

`ButtonGroup` visually joins several buttons into a segmented control. Props set on the group (`size`, `variant`, `isDisabled`, `fullWidth`) flow down to any child `Button` that doesn't override them.

<Sandbox code={`<ButtonGroup variant="bordered">
  <Button>Day</Button>
  <Button>Week</Button>
  <Button>Month</Button>
</ButtonGroup>`}>
  <ButtonGroup variant="bordered">
    <Button>Day</Button>
    <Button>Week</Button>
    <Button>Month</Button>
  </ButtonGroup>
</Sandbox>

Children can still override individual props — useful for highlighting the active option.

<Sandbox code={`<ButtonGroup variant="bordered">
  <Button>Day</Button>
  <Button variant="primary">Week</Button>
  <Button>Month</Button>
</ButtonGroup>`}>
  <ButtonGroup variant="bordered">
    <Button>Day</Button>
    <Button variant="primary">Week</Button>
    <Button>Month</Button>
  </ButtonGroup>
</Sandbox>

## API Reference

### Button

| Prop         | Type                                                                                                                        | Default     | Description                                         |
| ------------ | --------------------------------------------------------------------------------------------------------------------------- | ----------- | --------------------------------------------------- |
| `variant`    | `'primary' \| 'secondary' \| 'success' \| 'warning' \| 'danger' \| 'ghost' \| 'bordered' \| 'flat' \| 'shadow' \| 'light'`  | `'primary'` | Visual style + tone combined.                       |
| `size`       | `'sm' \| 'md' \| 'lg'`                                                                                                      | `'md'`      | Visual size.                                        |
| `isDisabled` | `boolean`                                                                                                                   | `false`     | Disables interaction and dims the button.           |
| `isIconOnly` | `boolean`                                                                                                                   | `false`     | Renders a square button for a single icon.          |
| `fullWidth`  | `boolean`                                                                                                                   | `false`     | Stretches to container width.                       |

Forwards all props of React Aria's `Button` — `onPress`, `onPressStart`, `onFocus`, etc. Native DOM event handlers (`onClick`) also work.

### ButtonGroup

| Prop         | Type                                   | Default     | Description                                              |
| ------------ | -------------------------------------- | ----------- | -------------------------------------------------------- |
| `variant`    | Button `variant` values                | —           | Default variant for unconfigured children.               |
| `size`       | Button `size` values                   | —           | Default size for unconfigured children.                  |
| `isDisabled` | `boolean`                              | —           | Disables all children that don't set their own value.    |
| `fullWidth`  | `boolean`                              | `false`     | Stretches the group to container width.                  |

Extends native `div` props. The group supplies defaults via context — any child `Button` that sets `size`, `variant`, `isDisabled`, or `fullWidth` wins over the group value.

## Accessibility

- Built on React Aria's `Button`, so the component handles `aria-disabled`, focus rings, and press events consistently across pointer and keyboard.
- Always provide an `aria-label` on `isIconOnly` buttons so screen readers announce their purpose.
- Prefer `isDisabled` over the native `disabled` attribute — it maps to the correct React Aria state and retains press-event consistency.
