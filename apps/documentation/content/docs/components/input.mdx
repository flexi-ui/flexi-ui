# Input

`Input` and `TextArea` are accessible text fields built on React Aria's `TextField`. Both share the same prop surface for labels, descriptions, and validation state, and `Input` adds two content slots for leading/trailing adornments like icons, prefixes, or inline actions.

## Installation

```bash
pnpm add @flexi-ui/input @flexi-ui/styles
```

## Import

```tsx
import { Input, TextArea } from '@flexi-ui/input'
```

## Error state

Pass `isInvalid` and `errorMessage` to signal a validation error. The message renders in the same slot as automatic browser errors, so transitions feel seamless.

<Sandbox code={`<Input
  label="Email"
  placeholder="you@example.com"
  isInvalid
  errorMessage="That address doesn't look quite right."
/>`}>
  <Input
    label="Email"
    placeholder="you@example.com"
    isInvalid
    errorMessage="That address doesn't look quite right."
  />
</Sandbox>

## API Reference

### Input (Root)

| Prop           | Type                                      | Default   | Description                                          |
| -------------- | ----------------------------------------- | --------- | ---------------------------------------------------- |
| `variant`      | `'flat' \| 'bordered' \| 'underlined'`    | `'flat'`  | Visual style.                                        |
| `size`         | `'sm' \| 'md' \| 'lg'`                    | `'md'`    | Visual size.                                         |
| `fullWidth`    | `boolean`                                 | `true`    | Stretches to container width.                        |
| `label`        | `string`                                  | —         | Label rendered above the field.                      |
| `description`  | `string`                                  | —         | Helper copy under the field.                         |
| `placeholder`  | `string`                                  | —         | Native placeholder.                                  |
| `errorMessage` | `string`                                  | —         | Explicit error copy. Falls back to native validation. |
| `isInvalid`    | `boolean`                                 | `false`   | Renders the error style.                             |
| `isDisabled`   | `boolean`                                 | `false`   | Disables the field.                                  |
| `startContent` | `ReactNode`                               | —         | Leading adornment inside the input row.              |
| `endContent`   | `ReactNode`                               | —         | Trailing adornment inside the input row.             |

Extends all props of React Aria's `TextField` — including `value`, `defaultValue`, `onChange`, `type`, `name`, `isRequired`, `pattern`, `minLength`, `maxLength`.

### TextArea (Root)

Same props as `Input` (minus `startContent` / `endContent`) plus:

| Prop   | Type     | Default | Description                  |
| ------ | -------- | ------- | ---------------------------- |
| `rows` | `number` | `3`     | Visible text line count.     |

## Accessibility

- Labels are always associated with the input via React Aria — no manual `htmlFor` needed.
- When `errorMessage` is set (or the browser reports a constraint error), it's announced via the proper `aria-describedby` relationship on the field.
- Prefer semantic `type` values (`email`, `url`, `tel`, `number`) so mobile keyboards and autofill pick the right experience.
- For fields without a visible label, include `aria-label` or use a visually hidden `<Label>` — never omit the label entirely.
