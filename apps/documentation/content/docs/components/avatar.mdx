# Avatar

The Avatar component displays user profile pictures, icons, or initials in a circular or rounded container. It's commonly used to represent users in your application with support for fallbacks, various sizes, colors, and interactive states.

## Import

```tsx
import { Avatar } from '@flexi-ui/avatar'
```

## Usage

### Basic Avatar with Image

Display a user avatar with a profile image:

<Sandbox code={`<Avatar
  src="https://i.pravatar.cc/150?u=a042581f4e29026024d"

/>`}>

    src="https://i.pravatar.cc/150?u=a042581f4e29026024d"

</Sandbox>

## API Reference

### Avatar Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | - | The image source URL for the avatar |
| `alt` | `string` | - | Alternative text for the image (important for accessibility) |
| `name` | `string` | - | Name to generate initials from when no image is provided |
| `size` | `"sm" \| "md" \| "lg" \| "xl"` | `"md"` | Size of the avatar component |
| `radius` | `"none" \| "sm" \| "md" \| "lg" \| "full"` | `"full"` | Border radius of the avatar (full creates a circle) |
| `color` | `"default" \| "primary" \| "secondary" \| "success" \| "warning" \| "danger"` | `"default"` | Color theme for the fallback background |
| `isBordered` | `boolean` | `false` | Whether to show a border around the avatar |
| `isDisabled` | `boolean` | `false` | Whether the avatar is in a disabled state (reduced opacity) |
| `showFallback` | `boolean` | `true` | Whether to show fallback content when image fails to load |
| `fallback` | `ReactNode` | - | Custom fallback content (icon, text, or component) |
| `classNames` | `AvatarClassNames` | - | Custom classes for component slots |
| `className` | `string` | - | Additional CSS class for the base element |
| `style` | `CSSProperties` | - | Inline styles for the base element |

### AvatarClassNames

The `classNames` prop accepts an object with the following keys:

| Key | Description |
|-----|-------------|
| `base` | Classes for the main avatar wrapper element |
| `img` | Classes for the image element |
| `fallback` | Classes for the fallback content container |
| `icon` | Classes for the fallback icon |

### Slots

The Avatar component is composed of the following slots:

- **base** - The main avatar wrapper that contains all elements
- **img** - The image element that displays the user's photo
- **fallback** - The fallback content container (displays initials or custom content)
- **icon** - The default fallback icon when no other fallback is provided

## Accessibility

The Avatar component follows accessibility best practices:

### ARIA Attributes

- Images include proper `alt` text for screen readers
- When using initials, they are properly announced by screen readers
- Disabled state is communicated through reduced opacity

### Keyboard Navigation

- When wrapped in interactive elements (buttons/links), avatars support full keyboard navigation
- Focus indicators are visible and meet WCAG contrast requirements
- Tab order is logical and predictable

### Screen Reader Support

- Image avatars use the `alt` prop for meaningful descriptions
- Initial-based avatars are announced with the user's name
- Status indicators should include `aria-label` for screen reader users

### Color Contrast

- Text initials maintain sufficient contrast against all background colors
- Border colors meet WCAG AA standards
- Disabled states maintain minimum contrast requirements

### Best Practices

1. **Always provide alt text** for image avatars to describe the person or content
2. **Use meaningful names** when generating initials for better screen reader experience
3. **Include status labels** when using status indicators for non-visual users
4. **Ensure interactive avatars** have visible focus states
5. **Test with keyboard navigation** to ensure all functionality is accessible

## Best Practices

### 1. Use Appropriate Sizes

Choose avatar sizes based on context:
- **sm**: Use in compact lists, tables, or inline mentions
- **md**: Default size for most use cases like comments and sidebars
- **lg**: Use in user profiles, headers, or prominent displays
- **xl**: Reserved for profile pages or hero sections

### 2. Provide Meaningful Alt Text

Always include descriptive alt text for images:

```tsx
// Good
<Avatar src="user.jpg" alt="John Doe, Software Engineer" />

// Avoid
<Avatar src="user.jpg" alt="avatar" />
```

### 3. Handle Missing Images Gracefully

Always have a fallback strategy:
- Use the `name` prop to generate initials
- Provide a custom fallback icon
- Enable `showFallback` to handle loading errors

### 4. Use Colors Consistently

Maintain color consistency across your application:
- Use `primary` for the current user or main subjects
- Use `default` for general users
- Reserve `danger` for deleted or blocked users
- Use colors semantically, not decoratively

### 5. Group Related Avatars

When showing multiple users:
- Use negative margins for overlapping groups
- Include borders to separate overlapping avatars
- Add a "+N" avatar to indicate additional users
- Limit visible avatars to 3-5 before showing count

### 6. Add Status Indicators Thoughtfully

When showing user status:
- Position indicators consistently (usually bottom-right)
- Use standard colors (green=online, yellow=away, gray=offline)
- Include ring borders to separate from avatar
- Provide accessible labels for status states

### 7. Make Interactive Avatars Clear

If avatars are clickable:
- Wrap in semantic elements (`<button>` or `<a>`)
- Add hover states to indicate interactivity
- Ensure focus states are visible
- Use appropriate cursor styles

### 8. Optimize Images

For best performance:
- Use appropriately sized images (avoid loading 4K images for small avatars)
- Implement lazy loading for avatar lists
- Consider using image CDNs with automatic optimization
- Use modern formats like WebP with fallbacks

### 9. Consider Loading States

Handle loading gracefully:
- Show skeleton loaders while images load
- Use the `showFallback` prop to handle errors
- Don't flash between loading and loaded states

### 10. Respect Privacy

Be mindful of user privacy:
- Allow users to opt for initials instead of photos
- Don't require profile pictures
- Provide default fallbacks that don't identify users
- Consider data protection regulations (GDPR, etc.)

## Troubleshooting

### Image Not Displaying

**Problem**: The avatar shows initials instead of the image.

**Solutions**:
1. Verify the image URL is correct and accessible
2. Check CORS settings if loading from external domains
3. Ensure the image format is supported (JPG, PNG, WebP, GIF)
4. Check browser console for loading errors
5. Verify the image isn't blocked by ad blockers or privacy extensions

```tsx
// Add error handling
<Avatar
  src="https://example.com/avatar.jpg"
  name="John Doe"  // Fallback to initials
  showFallback={true}
/>
```

### Initials Not Showing Correctly

**Problem**: Initials are missing or showing unexpected characters.

**Solutions**:
1. Ensure the `name` prop contains at least one word
2. Check for special characters in names
3. Verify the name isn't empty or undefined
4. Use proper capitalization in names

```tsx
// Correct usage
<Avatar name="John Doe" />  // Shows "JD"
<Avatar name="John" />      // Shows "J"

// Handle edge cases
<Avatar name={userName || "Guest User"} />
```

### Avatar Groups Overlapping Incorrectly

**Problem**: Avatars in groups don't overlap properly or have gaps.

**Solutions**:
1. Use negative margin classes (`-space-x-2`, `-space-x-4`, `-space-x-6`)
2. Adjust spacing based on avatar size
3. Ensure all avatars in group have the same size
4. Add `isBordered` to create visual separation

```tsx
// Correct grouping
<div className="flex -space-x-4">
  <Avatar src="..." size="md" isBordered />
  <Avatar src="..." size="md" isBordered />
  <Avatar src="..." size="md" isBordered />
</div>
```

### Status Indicator Position Issues

**Problem**: Status indicators are not positioned correctly.

**Solutions**:
1. Wrap avatar in a `relative` positioned container
2. Use `absolute` positioning for the indicator
3. Adjust position based on avatar size
4. Ensure proper z-index stacking

```tsx
// Correct status indicator
<div className="relative">
  <Avatar src="..." size="lg" />
  <span className="absolute bottom-0 right-0 h-3.5 w-3.5 rounded-full bg-green-500 ring-2 ring-white" />
</div>
```

### Blurry or Low-Quality Images

**Problem**: Avatar images appear blurry or pixelated.

**Solutions**:
1. Use higher resolution source images (at least 150x150px)
2. Ensure image dimensions match the display size
3. Check if CSS is scaling images incorrectly
4. Use 2x or 3x images for retina displays
5. Avoid using SVGs for photo avatars (unless intentional)

```tsx
// Use appropriate image sizes
<Avatar src="avatar-150.jpg" size="sm" />   // 32x32px display
<Avatar src="avatar-300.jpg" size="md" />   // 40x40px display
<Avatar src="avatar-600.jpg" size="xl" />   // 80x80px display
```

### Custom Styling Not Applied

**Problem**: Custom classes or styles aren't taking effect.

**Solutions**:
1. Use the `classNames` prop for slot-specific styling
2. Check CSS specificity conflicts
3. Verify Tailwind classes are not being purged
4. Use `!important` as a last resort
5. Ensure custom CSS is loaded after component styles

```tsx
// Correct custom styling
<Avatar
  src="..."
  classNames={{
    base: "ring-4 ring-purple-500",
    img: "object-cover object-top"
  }}
/>
```
