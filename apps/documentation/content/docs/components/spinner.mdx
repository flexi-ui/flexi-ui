# Spinner

`Spinner` is a loading indicator for asynchronous work. It renders an SVG with a subtle gradient, applies a `role="status"` so assistive tech knows work is in progress, and accepts an optional `label` for the status text.

## Installation

```bash
pnpm add @flexi-ui/spinner @flexi-ui/styles
```

## Import

```tsx
import { Spinner } from '@flexi-ui/spinner'
```

## API Reference

### Spinner

| Prop    | Type                                                                                            | Default     | Description                                                |
| ------- | ----------------------------------------------------------------------------------------------- | ----------- | ---------------------------------------------------------- |
| `size`  | `'sm' \| 'md' \| 'lg'`                                                                          | `'md'`      | Visual size.                                               |
| `color` | `'current' \| 'primary' \| 'secondary' \| 'success' \| 'warning' \| 'danger' \| 'white'`        | `'primary'` | Stroke color. `current` inherits the parent's text color.  |
| `label` | `string`                                                                                        | —           | Status text announced to screen readers and rendered beside the indicator. |

Extends native `svg` props (omits `display`, `opacity`, `color`, `size` — these are replaced by variant props).

## Accessibility

- Wrapped in a `<span role="status">` so assistive tech announces the loading state when the spinner appears.
- If the spinner is the only loading signal, always pass `label` — a silent spinner is invisible to screen readers.
- For long-running loads, consider pairing the spinner with a visible progress message so sighted users know what's being loaded.
