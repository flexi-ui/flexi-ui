# Form

`Form` is a thin wrapper around React Aria Components' `Form` primitive. It lays out its children, forwards native form events, and integrates with any field that supports the React Aria validation contract — including FlexiUI's own `Input` and `TextArea`.

## Installation

```bash
pnpm add @flexi-ui/form @flexi-ui/input @flexi-ui/button @flexi-ui/styles
```

## Import

```tsx
import { Form } from '@flexi-ui/form'
import { Input, TextArea } from '@flexi-ui/input'
import { Button } from '@flexi-ui/button'
```

## Usage

## Full width

Pass `fullWidth` to stretch the form and any descendant `fullWidth` fields to the container.

<Sandbox code={`<Form fullWidth onSubmit={(e) => e.preventDefault()}>
  <Input label="Subject" name="subject" />
  <TextArea label="Message" name="message" rows={4} />
  <Button type="submit">Send</Button>
</Form>`}>
  <Form fullWidth onSubmit={(e) => e.preventDefault()}>
    <Input label="Subject" name="subject" />
    <TextArea label="Message" name="message" rows={4} />
    <Button type="submit">Send</Button>
  </Form>
</Sandbox>

## Validation

`Form` participates in the standard HTML constraint validation API. FlexiUI's `Input` reads `isRequired`, `type`, `pattern`, and `minLength`/`maxLength` and surfaces browser messages via `FieldError` automatically.

<Sandbox code={`<Form onSubmit={(e) => { e.preventDefault(); alert('Valid — submitted') }}>
  <Input label="Email" name="email" type="email" isRequired />
  <Input label="PIN" name="pin" type="text" pattern="[0-9]{4}" isRequired description="4 digits" />
  <Button type="submit">Submit</Button>
</Form>`}>
  <Form onSubmit={(e) => { e.preventDefault(); alert('Valid — submitted') }}>
    <Input label="Email" name="email" type="email" isRequired />
    <Input label="PIN" name="pin" type="text" pattern="[0-9]{4}" isRequired description="4 digits" />
    <Button type="submit">Submit</Button>
  </Form>
</Sandbox>

## Custom error messages

Provide an `errorMessage` on any `Input` to display a server- or app-level error alongside the field.

```tsx
<Form>
  <Input
    label="Username"
    name="username"
    isInvalid
    errorMessage="This username is already taken."
  />
  <Button type="submit">Continue</Button>
</Form>
```

## API Reference

### Form

| Prop        | Type       | Default | Description                                     |
| ----------- | ---------- | ------- | ----------------------------------------------- |
| `fullWidth` | `boolean`  | `false` | Stretches the form to the container.            |

Extends all props of React Aria's `Form`, including `onSubmit`, `onReset`, `action`, `validationErrors`, and more. See the [React Aria Form docs](https://react-spectrum.adobe.com/react-aria/Form.html) for the complete list.

## Accessibility

- Renders a native `<form>` element, so screen readers announce it as a form landmark.
- Validation errors are wired through `FieldError` inside each `Input` — no manual `aria-describedby` plumbing required.
- Prefer native input types (`type="email"`, `type="url"`, etc.) so mobile keyboards and autofill services pick the right experience.
